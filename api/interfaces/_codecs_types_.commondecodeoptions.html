Optional asType

asType: keyof Types
  • Defined in src/codecs/types.ts:82

The node type to decode content to.

Many codecs decode to a single node type. e.g. the docx codec always decodes to an Article. However, some codecs can decode to multiple node types. e.g. the yaml and xlsx codecs. This option allows the user to be explicit about which node type they expect content to be decode to.

Optional format

format: undefined | string
  • Defined in src/codecs/types.ts:58

The format to decode content from.

Most codecs only decode from one format. However, for those codecs that support multiple formats, this options lets the user specify which one.

Optional isStandalone

isStandalone: undefined | false | true
  • Defined in src/codecs/types.ts:70

Should the content be treated as a standalone document or as a fragment?

This option affects whether codecs will attempt to extract metadata etc from the content. If this option is false, some codecs will default to returning an array of content nodes, instead of for example an Article node.
